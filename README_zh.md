# Commit Group

一个 VS Code 扩展,用于对 Git 修改文件进行分组管理,可同时维护多个 commit  分组的修改

## 功能特点

- 支持对 Git 修改文件进行分组管理
- 支持拖拽文件到不同分组
- 支持对分组进行重命名、删除等操作
- 支持对单个分组进行 commit 提交
- 自持对单个分组的多个文件 commit 提交
- 自动处理文件跟踪状态
- 支持导出分组文件列表
- 支持查看文件变更
- 支持多文件移动到其他分组

## 注意事项

- 需要工作区已初始化 Git 仓库
- 删除激活分组时,会弹框选择下一个激活的分组
- 合并冲突的文件会自动归类到特定分组
- 不支持对分组中的多个文件同时提交,防止误操作

## 安装

在 VS Code 扩展商店搜索 "Commit Group" 安装即可。
